# Contributing Guidelines

Thank you for considering contributing to Fredonbytes!

Please follow these guidelines when contributing:

- Fork the repository and create a feature branch for your changes using a descriptive name.
- Run a local build and ensure `npm run lint` and `npm run format` pass.
- Keep PRs small and focused on a single change or issue.
- Include a short summary for the PR describing the change, why it's needed, and any implementation details.
- Ensure your code follows existing project style and conventions.
- If you introduce breaking changes, include migration instructions and update the documentation.

Developer commands:

- Install: `npm install`
- Format: `npm run format`
- Lint: `npm run lint`
- Start Dev: `npm run dev`
- TypeCheck: `npm run type-check`
